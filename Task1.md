
# Практическое задание: Разработка системы управления персоналом (HRM Lite)

## 1. Общее описание

**Цель задания:** Закрепить навыки объектно-ориентированного программирования (ООП), работы с коллекциями (Java Collections Framework) и консольным вводом-выводом.

Вам необходимо разработать консольное приложение для управления данными сотрудников компании. Система должна позволять добавлять, удалять, искать и просматривать список сотрудников в оперативной памяти (In-Memory).

## 2. Технологический стек

* **Язык:** Java 21
* **Хранение данных:** Java Collections (List, Map)

---

## 3. Архитектура приложения

Проект должен быть разделен на логические пакеты:

1. `model` — описание сущностей (POJO классы).
2. `repository` — логика хранения данных.
3. `service` — бизнес-логика приложения.
4. `presentation` — взаимодействие с пользователем через консоль.

- Main class - HrmApplication

## 4. Подробные требования к реализации

### Шаг 1: Создание модели данных (`Employee`)

Создайте класс `Employee` со следующими полями:

* `Long id` — уникальный идентификатор (генерируется автоматически).
* `String name` — имя сотрудника.
* `String position` — должность (например: Developer, Manager, QA).
* `double salary` — заработная плата.
* `LocalDate hireDate` — дата приема на работу.

> **Важно:** Все поля должны быть приватными. Реализуйте геттеры, сеттеры, конструктор и метод `toString()`.

### Шаг 2: Слой хранения (`EmployeeRepository`)

Создайте класс, который будет имитировать базу данных.

* В качестве хранилища используйте `ArrayList<Employee>` или `HashMap<Long, Employee>`.
* Реализуйте методы:
* `save(Employee employee)` — сохранение нового сотрудника.
* `findAll()` — получение всех сотрудников.
* `findById(Long id)` — поиск по ID.
* `delete(Long id)` — удаление сотрудника.



### Шаг 3: Бизнес-логика (`HRMService`)

Создайте сервис, который управляет процессами:

* Расчет среднего значения зарплаты по всей компании.
* Поиск самого высокооплачиваемого сотрудника.
* Фильтрация сотрудников по должности.

### Шаг 4: Консольный интерфейс (`HRMApplication`)

В пакете presentatnion реализуйте бесконечный цикл `while` с текстовым меню:

1. Вывести список всех сотрудников.
2. Добавить нового сотрудника (данные вводятся через `Scanner`).
3. Удалить сотрудника по ID.
4. Поиск сотрудника по ID.
5. Показать статистику (средняя зарплата, топ-менеджер).
6. Выход.

## 6. Модульное тестирование (JUnit 5)

Чтобы убедиться, что ваша бизнес-логика работает правильно, необходимо написать **Unit-тесты**. Мы будем проверять слой `service`, так как именно там сосредоточены расчеты.

### Требования к тестам:

1. **Инструментарий:** Используйте библиотеку JUnit 5 
2. **Что тестируем:** Выберите 2-3 ключевых метода из `EmployeeService`.
3. **Изоляция:** Тесты не должны требовать ручного ввода данных в консоль.

### Пример того, что нужно проверить:

* **Успешный расчет:** Если в списке 3 сотрудника с зарплатами 100, 200 и 300, метод `calculateAverageSalary()` должен вернуть ровно 200.

## 7. Критерии оценки (Definition of Done)

1. **Корректность ООП:** Поля инкапсулированы, классы логически разделены.
2. **Работа с коллекциями:** Выбрана оптимальная коллекция для поиска и хранения.
3. **Обработка ошибок:** Программа не "падает" при вводе строки вместо числа (используйте `try-catch`).
4. **Чистота кода:** Переменные названы осмысленно (не `a`, `b`, `c`), соблюдены Java Naming Conventions.
